    • 完全背包问题(FullPack)

    问题模型

    有一个容量为WWW的背包和nnn种物品,第iii种物品的重量为wiw_iwi​价值为viv_ivi​,不同于01背包,每一种物品有无限件

    解析

    不同于01背包,每个物品有无限件,那么状态当然就不止取和不取两种,而是由不取,取1件,取2件,取3件……

    那么我们仍用01背包的状态,fi,jf_{i,j}fi,j​表示用前iii种物品,背包容量为jjj的最大价值

    转移时就需要枚举kkk表示取得件数,转移方程则为

    fi,j=max(fi−1,j−wi×k+k×vi)∣0≤k×wi≤j\Large f_{i,j}=max(f_{i-1,j-w_i \times k}+k \times v_i)| 0\leq k \times w_i \leq jfi,j​=max(fi−1,j−wi​×k​+k×vi​)∣0≤k×wi​≤j

    那么此时复杂度不是O(Wn)O(Wn)O(Wn)了,因为处理每一种物品需要W/wiW/w_iW/wi​的复杂度

    因为复杂度为O(n×∑i=1nW/wi)\large O(n \times \sum\limits_{i=1}^n W/w_i)O(n×i=1∑n​W/wi​)

    常数优化

    两种物品u,vu,vu,v满足wu≤wvw_u \leq w_vwu​≤wv​且Vu≥VvV_u\geq V_vVu​≥Vv​显然,我们可以直接去掉物品vvv,这一正确性是显然的,人家比你小,还比你有用

    这个优化需要O(n2)O(n^2)O(n2)处理,可以承受,但对于一些构造的数据,可能一种的去不掉,因此可以有另一种稍弱的方法

    去除重量高于WWW的物品,留下费用相同种价值最高的物品.O(W+n)O(W+n)O(W+n)

    更为高效转化

    不难想到,对于解析里的方法可以用另一种形式,将每个物品变为W/wiW/w_iW/wi​个独立的物品,直接变为01背包问题

    这样的复杂度依然与原来一样,但是可以考虑优化转化.

    利用二进制的思想,用集合S={2k∣k∈N+,k≤H}S=\{2^k|k∈N^+,k\leq H\}S={2k∣k∈N+,k≤H}中的若干个元素相加,可以表示出区间[1,2k+1−1][1,2^{k+1}-1][1,2k+1−1]中的所有整数.

    具体的,你可以用1,2,4,8,.....,2k1,2,4,8,.....,2^k1,2,4,8,.....,2k中若干个数的和表示出1,2,3....2k+1−11,2,3....2^{k+1}-11,2,3....2k+1−1

    那么对于每个物品ppp可以增加重量wp×2kw_p \times 2^kwp​×2k,价值vp×2k∣wi×2k≤Wv_p \times 2^k|w_i \times 2^k \leq Wvp​×2k∣wi​×2k≤W的独立物品,随后用01背包问题解决

    此时复杂度为Θ(n×log2W/wi)Θ (n \times log_2W/w_i)Θ(n×log2​W/wi​)已经很接近01背包的复杂度了

    最优解

    先看代码理解

    for (int i = 1; i <= n; i++)
    for (int j = w[i]; j <= W; j++)
        f[j] = max(f[j], f[j - w[i]] + v[i]);

    相比于01背包

    for (int i = 1; i <= n; i++)
    for (int j = W; j >= w[i]; j--)
        f[j] = max(f[j], f[j - w[i]] + v[i]);

    不同的就是jjj的枚举顺序,在01背包中,我们要确保更新fjf_jfj​时fj−w[i]f_{j-w[i]}fj−w[i]​还没被更新,一次你需要逆序枚举

    完全背包不同于01背包,因为他每种物品可以选无数次的性质,需要正序枚举jjj

    具体的,假设我们在第iii个物品,在我们更新fjf_jfj​时会用到fj−wif_{j-w_i}fj−wi​​,由于我们从小到大枚举的jjj,因此fj−wif_{j-w_i}fj−wi​​肯定已经被当前物品更新过了.这样就实现了选多个物品.(如果还没有理解,可以自己手绘表来模拟过程)

    那么代码只要从01背包小改以下就行了
